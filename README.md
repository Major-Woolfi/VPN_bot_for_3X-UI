# 🚀 VPN Telegram Bot (3X-UI)

Telegram‑бот для продажи и управления VPN‑подписками с интеграцией **3X‑UI Panel**. Поддерживает ручную оплату, подтверждение администратором, автоматическое создание/удаление клиентов в нескольких inbounds, учёт трафика, рефералку и пробный период. Управление и админ‑панель — прямо в Telegram.

Проект написан на **Python** и рассчитан на самостоятельный хостинг через **Docker**.

---

## ✨ Основные возможности

### 👤 Для пользователей

- Просмотр тарифов из `data/tarifs.json`
- Покупка VPN‑подписки и подтверждение оплаты
- Просмотр активной подписки и остатка трафика
- Получение ссылки подключения (sub URL)
- Уведомления об окончании подписки
- Пробный период (trial) для новых пользователей
- Реферальная система с бонусами

### 👑 Для администраторов

- Статистика по боту (пользователи, активные VPN, баны)
- Ручное подтверждение / отклонение платежей
- Тестовые подписки
- Блокировка и разблокировка пользователей
- Полный контроль подписок через Telegram

### 🔌 Интеграция с 3X‑UI

- Работа со всеми включёнными **inbounds**
- Создание клиента **в каждом inbound**
- Удаление клиента из всех inbounds по base email
- Подсчёт трафика как суммы по всем клиентам
- Авто‑реавторизация при протухших сессиях
- Поддержка VLESS / Trojan

## 💳 Логика оплаты

1. Пользователь выбирает тариф.
2. Переводит деньги вручную (карта).
3. Нажимает «Подтвердить оплату».
4. Запрос попадает в `data/await_payments.json`.
5. Администратор подтверждает или отклоняет платёж, а бот отправляет уведомление.

⚠️ **Важно:** комментарий к платежу обязателен только для поддержки (используется для опредения ID пользователя отправившего платёж).

| Премиум  | 300 ₽ | 10 | 2 ТБ   | 30 дней |

> ⚠️ Изменение тарифов **нужно проводить через `data/tarifs.json`**.

---

## 🔁 Логика работы с клиентами 3X‑UI

- Base email: `user_<telegram_id>@vpn.example`
- В каждый inbound добавляется клиент с **рандомным префиксом в 2 символа**
- При поиске / удалении используется base email (по окончанию адреса)
- Трафик суммируется по всем инбаундам

Это позволяет:

- обходить ограничения одного inbound
- корректно считать трафик
- безопасно пересоздавать клиентов

## ⏱ Фоновые задачи

### Проверка подписок (каждый час)

Бот проверяет `expiryTime`, удаляет клиентов при окончании и очищает подписку в БД.

### Очистка платежей (раз в 3 дня)

Удаляются старые обработанные платежи старше 30 дней.

## ▶️ Запуск (локально)

```bash
pip install -r requirements.txt
python main.py
```

Перед запуском убедитесь, что:

- `BOT_TOKEN` задан
- 3X‑UI доступен по сети
- логин/пароль/url панели корректны
- файл `data/tarifs.json` существует

## 🐳 Docker (упрощённый сценарий)

1. Откройте `Dockerfile` и `deploy_bot.sh` и замените путь к папке проекта на свой.
2. Запустите скрипт:

```bash
./deploy_bot.sh
```

Скрипт сам соберёт образ, установит зависимости и поднимет контейнер.

Важно: файлы `.env` и папка `data/` должны находиться внутри папки проекта.
