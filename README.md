# 🚀 VPN Telegram Bot (3X‑UI)

Telegram‑бот для продажи и управления VPN‑подписками с глубокой интеграцией в **3X‑UI Panel**.  
Поддерживает ручную оплату, модерацию платежей администраторами, автоматическое управление клиентами в нескольких `inbound`, систему лояльности (Trust Score), кастомные тарифы, реферальную программу и пробный период. Всё управление — прямо из Telegram.

Проект написан на Python 3.13+ и оптимизирован для самостоятельного хостинга через Docker.

---

## ✨ Основные возможности

### 👤 Для пользователей

- 📖 Просмотр фиксированных тарифов и **кастомных подписок** (гибкая формула цены)
- 💳 Покупка подписки с ручным переводом на карту и подтверждением оплаты
- 📊 Просмотр статуса подписки, остатка трафика, IP-лимитов и срока действия
- 🔗 Получение `sub URL` для импорта в клиенты (V2RayN, Sing-box, Nekoray и др.)
- 🎁 Пробный период (`trial`) для новых аккаунтов
- 🤝 Реферальная система: бонусные дни вам и приглашённому после первой оплаты
- 🏷️ Скидки до **50%** благодаря системе Trust Score

### 👑 Для администраторов

- 📈 Статистика: всего пользователей, активных VPN, заблокированных
- 💰 Модерация платежей с атомарным захватом и защитой от race-condition
- 🧪 Создание тестовых подписок без оплаты
- 🔨 Блокировка/разблокировка пользователей с указанием причины
- 📢 Массовые рассылки (всем зарегистрированным или только активным подписчикам)
- 🛠️ Инструменты отладки: ручное изменение Trust Score, нормализация подписок
- 🔐 Полный контроль через Telegram-интерфейс

### 🤖 Для системы

- 🔄 Автоматическая проверка и очистка истёкших/исчерпанных подписок (каждый час)
- 🧹 Автоудаление старых записей о платежах старше 30 дней (раз в 3 дня)
- 🔒 Автоматическое обновление SSL-сертификатов через SSH (каждые 5 дней)
- 💾 Надёжное хранение: SQLite для пользователей, JSON для очередей платежей

## 💳 Логика оплаты и Trust Score

1. Пользователь выбирает тариф → бот показывает реквизиты карты.
2. После перевода нажимает **«Подтвердить оплату»** → заявка попадает в `await_payments.json`.
3. Администратор проверяет поступление и нажимает `✅ Подтвердить` или `❌ Отклонить`.
4. При подтверждении:
   - Создаётся клиент в 3X-UI.
   - Начисляются очки Trust Score (`цена × 5%`).
   - Если есть реферер, выдаётся бонусный период.
   - Пользователь получает уведомление с `sub URL`.
5. **Trust Score** (0–100) даёт динамическую скидку до **50%** на следующие покупки. Штрафы начисляются при исчерпании трафика или отклонении платежа.

## 📦 Тарифы и Кастомные подписки

- Фиксированные тарифы задаются в `data/tarifs.json` (массив `plans` + `locations`).
- **Кастомный тариф** рассчитывается по формуле:
  `Итого = Base + (GB × GB_Coef) + (IP × Days × IP_Day_Coef) + Σ(Location_Day_Price × Days)`
- Бот автоматически сопоставляет выбранные локации с `remark`/`tag`/`name` включённых `inbound` в панели 3X-UI.

## 🔌 Интеграция с 3X‑UI

- Работает через REST API (`/panel/api/...`). Поддерживает авторизацию по **Bearer Token** (рекомендуется) или сессии.
- `Base email`: `user_<telegram_id>@vpn.com`
- При создании подписки клиент добавляется во все подходящие `inbound`.
- Трафик суммируется по всем клиентам одного `base_email`.
- При удалении/обновлении используется точное совпадение email, префиксы не мешают.
- Автоматический retry при `401/403` (протухшая сессия).

## ⏱ Фоновые задачи

| Задача | Частота | Описание |
|--------|---------|----------|
| `check_expired_subscriptions` | ~1 час | Проверяет `expiryTime` и лимиты трафика. Удаляет клиентов и чистит БД. |
| `cleanup_old_payments` | ~3 дня | Удаляет записи платежей со статусом `accepted/rejected` старше 30 дней. |
| `SSLUpdateTask` | ~5 дней | Подключается по SSH, останавливает nginx, вызывает `x-ui → 19 → 6` для перевыпуска сертификата, запускает nginx. Уведомляет админов. |
| `release_stale_processing` | При старте/остановке | Сбрасывает "зависшие" платежи из состояния `processing_*` обратно в `pending`. |

## 🚫 Блокировки и нормализация

- Заблокированные пользователи не могут взаимодействовать с ботом. Причина бана отображается сразу.
- Администраторов заблокировать невозможно.
- При блокировке Trust Score обнуляется, подписка удаляется.
- Команда **«Нормализация подписок»** восстанавливает клиентов, которые могли исчезнуть из `inbound` после рестарта панели, но остались в БД.

## ▶️ Запуск

### 🐧 Локально

```bash
pip install -r requirements.txt
cp .env.example .env  # отредактируйте
python main.py
```

### 🐳 Docker (рекомендуется)

1. Убедитесь, что `.env` и папка `data/` лежат рядом со скриптом.
2. Запустите:
```bash
chmod +x deploy_bot.sh
./deploy_bot.sh
```
3. Логи: `sudo docker logs -f vpn_bot`

> ⚠️ Скрипт `deploy_bot.sh` автоматически собирает образ, удаляет старый контейнер и запускает новый с `--restart always`.

## 📌 Ограничения и примечания

- 🔹 Платежи **не автоматизированы** (нет интеграции с платёжными шлюзами). Проверка вручную.
- 🔹 FSM хранится в памяти (`MemoryStorage`). При перезапуске бота активные диалоговые состояния сбрасываются.
- 🔹 Архитектура монолитная (`main.py`) для простоты деплоя и отладки.
- 🔹 Для стабильной работы API 3X-UI рекомендуется создать и использовать `PANEL_TOKEN` в настройках панели.
